00 / 기본 개념 잡기

왜 “데이터 중심”인가?

초기 딥러닝 연구의 불문율은 “공정한 비교(Fair Comparison)” — 같은 벤치마크 데이터셋을 고정해 두고 누가 더 좋은 모델을 만드는지 겨루는 것이었습니다. ResNet (CVPR 2016)이 대표적이죠. 하지만 현실 세계의 데이터는 고정되어 있지 않고, 지저분하며, 유명 테스트셋 10개에도 라벨 오류가 만연합니다. “Garbage in, garbage out” — 모델을 만지작거리는 것보다 데이터를 탐색하고 고치는 투자가 더 가치 있다는 것이 이 강의의 출발점입니다.

모델 중심 vs 데이터 중심 — 클릭해서 비교해 보세요

성능 = f( 모델 , 데이터(고정) )
  • 데이터셋은 주어진 것. 잘 정제된 벤치마크(예: 개/고양이 이미지)를 받아서 시작합니다.
  • 목표는 이 데이터셋에서 최고의 모델을 만드는 것 — 학교 ML 수업에서 배우는 방식입니다.
  • 도구: 신경망 구조 설계, 하이퍼파라미터 튜닝, 손실 함수 수정, 정규화(regularization)…
  • 한계: 기업과 사용자는 “정제된 데이터 위의 영리한 트릭”엔 관심이 없습니다. 현실 데이터에서 동작해야 하니까요.
성능 = f( 모델(고정) , 데이터 )
  • 모델은 무엇이든 좋다. 대신 훈련 데이터셋을 체계적·알고리즘적으로 개선합니다.
  • 형태 ① 데이터를 이해해서 모델을 개선 — 예: (쉬운 데이터부터 학습)
  • 형태 ② 데이터를 수정해서 모델을 개선 — 예: (잘못 라벨된 데이터를 훈련 전에 제거)
  • 최신 트렌드: DeepSeek의 학습 데이터 밈, DALL·E 2의 데이터 이슈, “모델은 공개해도 데이터는 숨기는” Molmo가 지적한 LVLM 개방성 문제처럼, 이제 경쟁력의 원천은 데이터입니다.

02 / Towards Automatic Data Generation

생성 모델로 훈련 데이터를 만들다

ChatGPT 같은 , Sora 같은 영상 생성기 등 가 충분히 강력해지자 발상이 뒤집힙니다 — 생성 모델을 “데이터 생성 에이전트”로 쓰자. 실제로 SynCLR (CVPR 2024)은 합성 이미지만으로 학습한 모델이 실제 데이터 학습에 필적함을 보였고, CVPR에는 합성 데이터 전용 워크숍 (SynData4CV, SynTagen)이 생겼습니다.

합성 데이터의 스케일링 — 무엇이 성능을 좌우하나

Scaling Laws of Synthetic Images (CVPR 2024)에 따르면, 생성 데이터의 효과는 3가지 요인이 좌우합니다. 특히 이미지-텍스트 간 “개념 불일치(concept mismatch)”가 적을수록 잘 스케일됩니다.

① 어떤 T2I 모델인가

Stable Diffusion / Imagen / Muse 등 의 선택 자체가 데이터 품질을 결정합니다.

② Guidance Scale

을 낮추면 더 다양한 이미지가 생성되고 → 학습 성능이 좋아집니다.

③ 텍스트 프롬프트

프롬프트가 다양할수록 이미지도 다양해지고 → 성능이 좋아집니다. 다양성이 핵심 화폐입니다.

발표자의 연구: 서술 기반 객체 탐지를 위한 데이터 자동 생성

는 “수영장에 뛰어드는 사람을 찾아줘”처럼 자유 문장으로 객체를 찾는 과제입니다. 이를 위해선 비전(위치 찾기) + 언어(문장 이해) + 멀티모달(연결) 세 가지 지식이 모두 필요한데, 기존 모델(GLIP·FIBER)은 설명이 길어질수록 성능이 급락했습니다 (GLIP: 짧은 설명 29.4 → 긴 설명 8.2). 원인은 “짧고 빈약한 설명만 담긴 기존 데이터”. 해법으로 발표자는 흩어져 있는 AI 모델들의 지식을 하나로 모으는 지식 통합 학습(Knowledge Consolidated Learning) 프레임워크를 제안합니다. 03 / Quality vs Quantity

품질 vs 수량 — 어떤 데이터가 더 좋을까?

KOALA (NeurIPS 2024)는 로 SDXL을 압축해 1024×1024 이미지를 1초 만에 생성하는 경량 확산 모델입니다. 이 연구 과정에서 얻은 “훈련 데이터에 관한 경험적 교훈”이 흥미롭습니다. 직접 맞혀 보세요 — 어느 쪽이 더 좋은 훈련 데이터였을까요?

라운드 1 — 캡션의 출처

vs
정답: B. 이미지는 동일해도 캡션의 정보 밀도가 다릅니다. 인터넷 크롤링 텍스트(alt 태그 등)는 짧고 부정확한 반면, 캡셔닝 모델이 생성한 72단어 설명은 이미지 내용을 충실히 담아 텍스트-이미지 정렬 학습에 훨씬 유리합니다. “데이터를 고치는 가장 싼 방법은 라벨을 다시 쓰는 것”의 좋은 예시입니다.

라운드 2 — 800만 장 vs 49만 장

vs
정답: B. 수량이 16배 적어도, 고해상도 + 고품질 캡션의 49만 장이 이겼습니다. 같은 교훈이 다른 분야에서도 반복됩니다 — 영상 생성의 Stable Video Diffusion은 데이터 큐레이션이 성능을 갈랐고, LLM 추론의 s1 (test-time scaling)은 엄선된 단 1,000개 샘플로 추론 능력을 끌어올렸습니다. “현재로서는, 데이터 품질이 수량보다 중요해 보인다.”

CaptionSmiths — 데이터로 만든 “슬라이더 제어”를 체험해 보세요

CaptionSmiths (ICCV 2025 Highlight)는 캡션의 길이·서술성·고유성을 슬라이더처럼 연속적으로 제어하는 모델입니다. 비결은 모델 구조가 아니라 데이터 중심 해법 — 훈련 캡션마다 규칙 기반 계산기로 길이/서술성/고유성 점수를 매겨 조건으로 붙여 학습시킨 것뿐입니다. 아래 슬라이더로 그 느낌을 체험해 보세요.

04 / Future Problems

미래 과제 — 스스로 진화하는 AI를 향해

지금까지의 과정(생성 → 학습 → 평가 → 피드백)에는 여전히 사람이 끼어 있습니다. 다음 단계는 가 이 사이클 전체를 자율적으로 굴리는 “완전 자동 데이터 구축 + 모델 학습 프레임워크 = 자기 진화(Self-evolving) AI”입니다.

에이전트가 굴리는 데이터 루프
  • 🔍 검색(Retrieve) — 관련 데이터를 기존 소스에서 찾아오고
  • 🎨 생성(Generate) — 생성 모델로 부족한 데이터를 만들고
  • 🕹️ 시뮬레이션(Simulate) — 자동 시뮬레이션으로 검증 환경을 구성하고
  • 📊 평가·피드백(Evaluate & Feedback) — 실패를 분석해 다시 데이터로
초기 연구 사례와 한계

Evaluation Agent는 T2I 확산 모델을 에이전트가 효율적으로 평가하는 프레임워크지만, 아직 평가에만 적용되고 에이전트 설계가 쉬운 T2I 분야에만 국한된 초기 단계입니다. 전체 사이클의 자동화 — 그것이 열려 있는 연구 기회입니다.
