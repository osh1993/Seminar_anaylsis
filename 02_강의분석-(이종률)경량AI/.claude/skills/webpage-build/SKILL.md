# 세미나 해설 웹페이지 빌드 명세

## 산출 구조
```
webpage/
├── index.html   # 뼈대 + 안내
├── style.css    # 스타일 (다크/라이트 무관, 가독성 우선)
├── app.js       # 렌더링 + 검색 + Q&A 로직
├── data.js      # window.SEMINAR_DATA = {...}  (day별 분할 가능: data_day1.js 등)
└── slides/      # _workspace/slides 복사 (robocopy 사용)
```
서버 없이 `file://`로 열려야 하므로 `fetch()`로 JSON을 읽지 말고 `data.js`에 전역 변수로 심는다. 외부 CDN 금지.

## data.js 생성
Python 스크립트로 `_workspace/01_extract_*.json` + `02_interpret_*.json` + `03_references.json`을 병합해 생성한다 (수작업 금지):
```json
{
  "meta": {"title": "...", "days": [{"day":1,"title":"...","pages":213}]},
  "slides": {"d1p5": {"day":1,"page":5,"image":"slides/day1/005.png",
              "text":"원문","section":"...","title":"...","one_liner":"...",
              "explanation":"...","key_terms":[...],"difficulty":"...",
              "references":[...],"related_slides":[...],"qa":[...]}},
  "sections": [{"day":1,"name":"Background","pages":[4,30]}],
  "references": [ 03_references.json 내용 ]
}
```
슬라이드 원문 text도 포함한다(검색 대상). 병합 스크립트는 `_workspace/build_data.py`로 저장해 재실행 가능하게 한다.

**강의 실황 오버레이 병합:** `_workspace/05_lecture_day{N}_*.json`(강의 녹취록 기반 서사)이 있으면 병합해 각 슬라이드에 `lecture_note`(문자열) 필드를 추가하고, 오버레이의 `lecture_qa`는 슬라이드 `qa` 배열에 `from_lecture:true` 표식과 함께 이어붙인다. explanation(슬라이드 근거)과 lecture_note(강사 구술 서사)는 별개 필드로 유지한다.

## 화면 구성 (뷰 탭)
추가 뷰(강의 종합):
- **⭐ 강조**(`view-highlights`): `06_highlights_day{N}.json`을 카테고리(핵심 기법/실무 조언/주의·함정/강사 평가/산업 인사이트)별로 그룹핑, 카테고리 필터 버튼 + high 우선 정렬 + 슬라이드 점프 링크. 데이터는 `D.highlights`.
- **🧭 연구 흐름**(`view-flow`, 인터랙티브): `07_flow_day{N}.json`. 상단 갈래(track) 탭으로 한 갈래씩 선택 → 세로 타임라인. 각 step은 접힘 상태에서 label+`concept`(한 줄 개념), 클릭 시 `note`+슬라이드 링크 펼침. step 사이 `transition`(이전 한계→다음 등장 이유)을 화살표 다리로 표시. "다음 개념 ▶/◀ 이전"으로 단계별 따라가기(focus 하이라이트+스크롤), "모두 펼치기/접기". 데이터: `D.flow`(`intro`+`tracks[].steps[{label,concept,transition,note,pages,era}]`). concept/transition은 요약 레이블이라 인용 태그 없음(근거는 note·pages).

## 화면 구성 (4개 뷰 탭)
1. **슬라이드 뷰** (기본): 좌측 목차(day > 섹션 > 슬라이드), 중앙 슬라이드 이미지, 우측 해설 패널(한줄요약·해설·**🎙️ 강의 실황**·핵심용어·참조문헌·관련슬라이드·확인질문). 해설(슬라이드 근거) 바로 아래 강의 실황(lecture_note) 블록을 앰버 톤으로 구분 렌더. 강의 중 실제 문답(qa.from_lecture)은 "강의 중 질문" 배지로 표시. ←/→ 키보드 내비게이션. 인용 태그 [D1-p23]는 `<a>` 링크로 변환해 클릭 시 해당 슬라이드로 점프.
2. **검색**: 입력어를 공백 분리해 제목/원문/해설/강의실황/용어를 대상으로 점수화(제목 3점, 용어 2점, 해설 1.5점, 강의실황 1.5점, 원문 1점, AND 우선). 결과는 슬라이드 카드(썸네일+하이라이트 스니펫)로, 클릭 시 슬라이드 뷰로 이동. 초성/부분일치까지는 불요, 대소문자 무시 필수.
3. **Q&A**: 질문 입력 → (a) 해설 데이터의 qa 배열에서 유사 질문 매칭(단어 겹침 점수), (b) 검색 점수 상위 슬라이드의 해설 결합. 답변에는 반드시 출처 슬라이드 링크를 표기하고, 상단에 "이 답변은 세미나 자료 검색 기반으로 자동 구성됩니다" 안내를 고정 표시한다. 자주 묻는 질문(수집된 qa 중 대표 10~15개)을 버튼으로 노출.
4. **참고문헌**: 03_references 목록을 유형별로 표시, 각 항목에 인용 슬라이드 링크(cited_by).

## 부가 기능 (슬라이드 뷰)
1. **뷰어 크기 조절**: 좌측 목차↔중앙 슬라이드, 중앙↔우측 해설 패널 사이에 드래그 스플리터를 둔다. 드래그로 폭 조절, 더블클릭 시 기본값 복원, 조절값은 localStorage에 저장해 재방문 시 유지. 각 영역에 최소 폭을 두어 붕괴 방지.
2. **슬라이드별 메모**: 우측 해설 패널에 슬라이드별 메모 입력란(textarea)을 둔다.
   - 자동 저장: 입력 중 debounce 후 localStorage(`memo:d{day}p{page}`) 저장, 저장 상태 표시
   - 목차에 메모 있는 슬라이드 표시(점 배지)
   - "내 메모" 모아보기: 전체 메모를 슬라이드 링크와 함께 나열하는 뷰(또는 패널), 마크다운(.md) 파일로 내보내기 버튼 제공
   - **내보내기/가져오기 왕복(round-trip)**: 내보내는 .md에 슬라이드 키를 파싱 가능한 형식(예: `## [D1-p23] 제목`)으로 포함하고, "가져오기" 버튼(파일 선택)으로 그 .md를 읽어 localStorage에 복원한다 — 다른 브라우저/PC로 메모 이동 경로. 동일 슬라이드에 기존 메모가 있으면 사용자 확인 후 덮어쓰기, 가져오기 결과(복원 n건)를 표시
   - localStorage는 브라우저별 저장임을 UI에 짧게 안내하고, 옮기려면 내보내기→가져오기를 쓰라고 함께 안내
   - 검색 기능이 메모 내용도 검색 대상에 포함하면 좋다(선택)

## 필수 동작 규칙
- 인용 태그 변환 정규식: `\[D(\d)-p(\d+)\]` → 슬라이드 점프 링크. 변환 누락 0건.
- (배경지식) 마커는 회색 배지로 렌더링해 슬라이드 근거와 시각적으로 구분한다.
- URL 해시 라우팅(`#d1p23`)으로 특정 슬라이드 딥링크 지원.
- 이미지 지연 로딩(`loading="lazy"`) — 슬라이드 440장이므로 필수.
- 해설 누락 슬라이드는 "해설 준비 중" 배지 + 원문 텍스트 표시.

## 검증 (빌드 후 자체 수행)
- data.js의 슬라이드 수 = `01_extract_day*.json` 슬라이드 수 합계 (현재 467 = day1 221 + day2 246).
- `node --check app.js && node --check data.js` (node 없으면 생략하고 보고).
- 인용 태그 원본 수 = 변환 후 링크 수 (Python으로 카운트 대조).
