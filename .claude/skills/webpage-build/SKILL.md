# 세미나 해설 웹페이지 빌드 명세

## 산출 구조
```
{세미나폴더}/webpage/
├── index.html   # 뼈대 + 안내
├── style.css    # 스타일
├── app.js       # 렌더링 + 검색 + Q&A + 개념사전 로직
├── data.js      # window.SEMINAR_DATA = {...}  (doc별 분할 가능: data_day1.js 등)
└── slides/      # _workspace/slides 복사 (robocopy 사용)
```
서버 없이 `file://`로도, GitHub Pages에서도 열려야 하므로 `fetch()`로 JSON을 읽지 말고 `data.js`에 전역 변수로 심는다. 모든 경로는 상대 경로. 외부 CDN 금지(웹폰트는 시스템 폰트 폴백 필수 조건으로 허용).

**참고 구현**: `02_강의분석-(이종률)경량AI/webpage/`가 이 명세의 완성 선례다. 새 세미나 빌드 시 그 구조·스타일을 출발점으로 삼되, 자료 성격에 맞게 조정한다.

## data.js 생성
Python 스크립트로 `_workspace/`의 01_extract + 02_interpret + 03_references + (05_lecture) + 06_highlights + 07_flow + 08_concepts를 병합해 생성한다 (수작업 금지). 병합 스크립트는 `_workspace/build_data.py`로 저장해 재실행 가능하게 한다.
```json
{
  "meta": {"title": "...", "days": [{"day":1,"title":"...","pages":213}]},
  "slides": {"d1p5": {..., "lecture_note": "오버레이 있으면", "concept_ids": ["pruning"]}},
  "sections": [{"day":1,"name":"Background","pages":[4,30]}],
  "references": [...], "highlights": [...], "flow": {...},
  "concepts": [08_concepts_*.json 병합, id 중복 시 나중 것 채택·보고]
}
```
- 슬라이드 원문 text도 포함한다(검색 대상).
- `05_lecture_*`가 있으면 슬라이드에 `lecture_note` 추가, `lecture_qa`는 qa 배열에 `from_lecture:true`로 이어붙인다.
- `concept_ids`: 슬라이드의 key_terms/본문에 개념 사전 항목(aliases 포함)이 등장하면 매핑 — 해설 옆 "개념 더 보기" 링크의 데이터.

## 상단 네비게이션 (모든 세미나 페이지 공통, 필수)
- 헤더에 **"← 세미나 목록"** 링크를 상시 표시한다 — 허브로 복귀. `webpage/` 하위 페이지에서는 `../../index.html`, 폴더 직하 단일 파일(01 유형)에서는 `../index.html`. 절대 경로 금지.
- 두 모드가 모두 있는 세미나는 **모드 전환 링크**를 함께 표시: 슬라이드 해설(`index.html`) ↔ 📖 강의 노트(`note.html`).

## 📖 강의 노트 모드 (note.html) — 슬라이드 뷰와 공존
`_workspace/09_note.json`(slide-interpretation 스킬의 note 스키마)을 렌더한 **단일 파일**(CSS/JS 인라인, 데이터도 인라인 — fetch 금지). 디자인은 `01_fable_강의분석_Data-Centeric AI/index.html` 계승: 다크 종이 질감, 세리프 제목, 이어 읽는 긴 글 레이아웃.
- 구성: 헤더(제목·강사·읽기 시간·상단 네비) → 프롤로그 → 장 목차(진행 표시, 클릭 점프) → 장별 서사(문단 + key_points 박스 + "이 구간 슬라이드로 보기 → `index.html#d1p{시작페이지}`") → 에필로그.
- 인용 태그 [D1-pN]는 `index.html#d1p{n}` 링크(작은 배지)로 변환 — 노트에서 슬라이드 뷰의 해당 장면으로 점프하는 통로다. `(배경지식)`/`(모델 지식)` 마커는 회색 배지.
- 간단 마크다운 렌더: `**굵게**`, `- ` 목록, `\n\n` 문단.
- 슬라이드 이미지는 넣지 않는다(가볍게 유지) — 이미지는 슬라이드 뷰가 담당.

## 화면 구성 (뷰 탭)
1. **슬라이드 뷰** (기본): 좌측 목차(doc > 섹션 > 슬라이드), 중앙 슬라이드 이미지, 우측 해설 패널(한줄요약·해설·🎙️강의실황·핵심용어·**개념 더 보기 링크**·참조문헌·관련슬라이드·확인질문). ←/→ 키보드 내비게이션. 인용 태그 [D1-p23]는 링크로 변환, 클릭 시 해당 슬라이드로 점프.
2. **검색**: 공백 분리 점수화(제목 3, 용어 2, 해설 1.5, 강의실황 1.5, 개념사전 1.5, 원문 1, AND 우선). 결과는 카드(썸네일+하이라이트 스니펫), 대소문자 무시.
3. **📚 개념 사전** (`view-concepts`): `D.concepts`를 카테고리별 그룹핑. 좌측 개념 목록(가나다/카테고리), 우측 상세(easy_def → detail → in_this_seminar → misconceptions → sources 링크 → related_concepts 체인). `(모델 지식)` 마커는 회색 배지 렌더. 딥링크 `#concept-{id}`. 해설 패널의 용어에서 이 뷰로 점프.
4. **⭐ 강조** (`view-highlights`): 카테고리 필터 버튼 + high 우선 정렬 + 슬라이드 점프 링크.
5. **🧭 연구 흐름** (`view-flow`, 인터랙티브): 상단 갈래(track) 탭 → 세로 타임라인. step 접힘 상태에서 label+concept, 클릭 시 note+슬라이드 링크 펼침. step 사이 transition 화살표 다리. "다음 개념 ▶/◀ 이전" 단계별 따라가기(focus+스크롤), "모두 펼치기/접기".
6. **Q&A**: 질문 입력 → qa 배열 유사 질문 매칭(단어 겹침) + 검색 상위 슬라이드 해설 결합. 답변에 출처 슬라이드 링크 필수, "세미나 자료 검색 기반 자동 구성" 안내 고정. 대표 질문 10~15개 버튼 노출.
7. **참고문헌**: 03_references를 유형별 표시, 각 항목에 인용 슬라이드 링크(cited_by), verified_manual은 "검증 등재" 배지.

## 부가 기능 (슬라이드 뷰)
1. **뷰어 크기 조절**: 목차↔슬라이드↔해설 패널 사이 드래그 스플리터. 더블클릭 기본값 복원, localStorage 저장, 최소 폭 보장.
2. **슬라이드별 메모**: textarea, debounce 자동 저장(localStorage `memo:d{day}p{page}`), 목차 점 배지, "내 메모" 모아보기, `.md` 내보내기/가져오기 왕복(헤딩 키 `## [D1-p23] 제목` 파싱, 덮어쓰기 확인, 복원 건수 표시). localStorage가 브라우저별 저장임을 UI에 안내.

## 디자인 원칙 (휴먼 친화)
- 따뜻한 색조·넉넉한 행간·명확한 위계. 참고 팔레트: `01_fable_강의분석_Data-Centeric AI/index.html`(다크 종이 질감), `02_.../webpage/style.css`.
- 첫 화면에 "이 세미나는 무엇인가 + 어떻게 읽으면 되는가" 안내 카드를 둔다 — 학습자가 연구 흐름 → 슬라이드 → 개념 사전 순으로 진입하도록 유도.
- 모바일에서도 읽히게 반응형 최소 대응(목차 접기).

## 필수 동작 규칙
- 인용 태그 변환 정규식: `\[D(\d+)-p(\d+)\]` → 슬라이드 점프 링크. 변환 누락 0건.
- `(배경지식)`·`(모델 지식)` 마커는 회색 배지로 렌더링해 슬라이드 근거와 시각적으로 구분.
- URL 해시 라우팅(`#d1p23`, `#concept-{id}`) 딥링크 지원.
- 이미지 지연 로딩(`loading="lazy"`) 필수.
- 해설 누락 슬라이드는 "해설 준비 중" 배지 + 원문 텍스트 표시.

## 허브 인덱스 갱신 (루트 index.html)
새 세미나 빌드 완료 시 저장소 루트 `index.html`의 세미나 카드 목록에 항목을 추가한다: 제목, 한 줄 소개, 발표자/일자(알 때만), 슬라이드 수. **카드에는 존재하는 모드별 진입 버튼을 단다**: "🎞️ 슬라이드 해설"(`{폴더}/webpage/index.html`), "📖 강의 노트"(`{폴더}/webpage/note.html`, 01 유형은 단일 index.html). 기존 카드는 건드리지 않는다. 폴더명의 공백·특수문자는 href에서 URL 인코딩한다(공백 %20, & %26).

## 검증 (빌드 후 자체 수행)
- data.js 슬라이드 수 = `01_extract_day*.json` 슬라이드 수 합계, 개념 수 = 08_concepts 합계.
- `node --check app.js && node --check data.js` (node 없으면 생략하고 보고).
- 인용 태그 원본 수 = 변환 후 링크 수 (Python으로 카운트 대조).
