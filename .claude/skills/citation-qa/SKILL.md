# 인용 정합성 QA 방법론

## 원칙
"파일이 있다"는 검증이 아니다. **경계면 양쪽을 동시에 읽고 내용을 맞대는 것**이 검증이다.
검증 스크립트와 결과는 `{폴더}/_workspace/qa/`에 남긴다 (재실행·감사 추적).

## 검사 1: 인용 태그 전수 검사 (스크립트)
`02_interpret_*.json`, `05_lecture_*.json`, `06/07/08_*.json`의 모든 텍스트 필드에서 `\[D(\d+)-p(\d+)\]`를 추출해:
- 유효 doc·페이지 범위는 하드코딩하지 말고 **`01_extract_day{N}.json`의 슬라이드 수를 읽어** 판정 → 위반은 **치명**
- 태그도 `(배경지식)`/`(모델 지식)` 마커도 없는 explanation 문장 비율 계산 → 10% 초과 시 **중대**

## 검사 2: 인용 내용 표본 대조 (모델 판단)
doc별 10개 이상 슬라이드를 균등 간격 추출하여, 해설과 원문 text(필요시 PNG)를 비교:
- 해설의 주장이 원문에 실제로 있는가? 없으면 **치명** (환각)
- 수치/기법명이 원문과 일치하는가? 불일치는 **중대**

## 검사 3: 참조 URL 전수 대조 (스크립트)
`02_*.json`의 references.url과 `03_references.json`의 url이 `01_extract_*.json`의 urls 합집합에 존재하는지 확인. 원문에 없는 URL은 **치명** (URL 환각). 대소문자/말미 구두점은 정규화 후 비교.
예외: `verified_manual: true` 항목은 원문에 없어도 허용 — 단, `evidence`에 검증 근거가 기록되어야 한다. 근거 없는 verified_manual은 **중대**.

## 검사 4: 개념 사전 검사 (표본 + 스크립트)
- `08_concepts_*.json`의 id 중복, in_this_seminar 태그 범위 → 스크립트 전수
- 표본 5개 개념: sources URL이 실제 해당 개념을 다루는지 접속 확인(WebFetch), `(모델 지식)` 마커 없는 핵심 주장에 source가 붙어 있는지 → 위반은 **중대**

## 검사 5: 웹 데이터 정합성 (스크립트)
- data.js 슬라이드 수 = 01_extract 합계, 해설 수 = 해설 JSON 합계, 개념 수 = 08 합계
- 해설 JSON의 explanation과 data.js의 explanation 무작위 5건 문자열 일치
- 인용 태그가 링크로 변환되는지: app.js에 변환 정규식 존재 + 태그 수 대조

## 검사 6: 웹 기능 정적 검증
- `node --check`로 JS 문법 (node 없으면 Python으로 괄호 균형이라도)
- index.html의 검색 input, Q&A input, 뷰 탭 id가 존재하고 app.js가 해당 id를 참조하는지 교차 확인
- 이미지 경로 표본 10건이 실제 파일로 존재하는지

## 검사 7: 게시 전 점검 (게시 직전 요청 시)
- webpage 내 모든 로컬 참조(src/href)가 상대 경로인지, 절대 경로(`file://`, `D:\`, 루트 `/`)가 없는지 전수 검사 → 위반은 **치명** (Pages에서 깨짐)
- 참조 경로의 실제 파일 존재 + **대소문자 정확 일치** (GitHub Pages는 구분) → 위반은 **치명**
- 루트 허브 index.html의 세미나 링크가 실제 폴더/파일과 일치하는지
