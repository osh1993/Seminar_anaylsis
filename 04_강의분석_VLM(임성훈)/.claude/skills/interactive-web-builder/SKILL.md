# Interactive Web Builder — 교육 웹페이지 빌드 스킬

백엔드 없이 브라우저에서 바로 열리는 단일 페이지 교육 웹앱을 만든다.

## 사이트 구조
```
site/
├── index.html
├── css/style.css
├── js/
│   ├── data.js      # 모든 콘텐츠 JSON을 window 전역으로 인라인 (file:// fetch 제약 회피)
│   ├── app.js       # 라우팅/렌더/인터랙션
│   └── qa.js        # Q&A 검색 + FAQ
├── data/            # 원본 JSON (감사/재사용용 사본)
└── assets/slides/   # 슬라이드 PNG
```

## 필수 인터랙티브 요소
1. **기술 흐름 구성도(Hero Flow Map)** — flow.json 기반. 노드 클릭 → 해당 섹션으로 스크롤/이동. 두 트랙(자료1 연구 / 자료2 VLM) + 공유 개념 교차 링크.
2. **개념 카드** — headline 항상 노출, 클릭 시 intuition/explanation/why 펼침. 각 카드 하단에 출처 배지(`📄 자료2 · p17`)와 논문 칩(클릭→arXiv).
3. **탭/필터** — 자료1 / 자료2 / 통합 흐름, 주제별 필터.
4. **슬라이드 라이트박스** — 그림 썸네일 클릭 시 확대, 캡션·출처 표시.
5. **Q&A 패널** — 입력창 + FAQ 칩. 동작: (a) FAQ 정답 우선, (b) 콘텐츠 전 카드 대상 키워드/토큰 매칭 검색 → 상위 결과를 출처와 함께 카드로 제시. LLM 연결용 `askLLM()` 훅을 주석으로 남긴다.
6. **용어 툴팁** — 핵심 용어에 hover 풀이.

## 출처 표기 규율 (사용자 핵심 요구)
- 모든 설명 블록에 슬라이드 참조 배지 + 관련 논문 링크. 출처 없는 단정 금지.
- 논문 칩은 references.json의 url로 연결, url 없으면 제목 검색 링크(google scholar) 또는 비활성 표시.

## 디자인
- frontend-design 스킬 원칙 적용. 학술적 신뢰감 + 가독성. 명확한 정보 위계, 절제된 색, 부드러운 모션.
- 반응형, 키보드 접근, 이미지 alt, 다국어 불필요(한국어 기본).

## 데이터 로딩 주의
- 로컬 더블클릭(file://)으로도 열리도록 콘텐츠는 `js/data.js`에 `window.SITE_DATA = {...}` 형태로 인라인한다. fetch는 http 서버에서만 동작하므로 보조 경로로만.
