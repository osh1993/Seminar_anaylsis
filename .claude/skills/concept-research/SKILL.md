# 개념 심층 리서치 규칙

## 목표
학습자가 슬라이드 해설(요점)에서 한 단계 더 들어가 **개념을 상세하게 이해**할 수 있는 개념 사전을 만든다. 각 항목은 (1) 쉬운 정의 → (2) 원리 상세 → (3) 세미나 맥락 → (4) 더 읽을거리(검증된 링크) 구조를 갖는다.

## 항목 선정
- 오케스트레이터가 해설 JSON의 `deep_dive_candidates`를 집계·중복 제거해 분배한다. 세미나당 15~40개 수준.
- 같은 개념의 표기 변형(Pruning/프루닝/가지치기)은 하나로 통합하고 `aliases`에 기록한다 — 웹 검색·해설 링크 매칭에 쓰인다.

## 리서치 원칙
1. **검색 → 확인 → 기록**: 정의·제안 논문·핵심 수치는 WebSearch로 찾고 WebFetch로 원문(초록 수준)을 확인한 뒤 기록한다. 확인한 URL을 `sources`에 남긴다.
2. **URL 조립 금지**: arXiv 번호·DOI를 기억으로 만들지 않는다. 검색 결과에 나온 URL만 쓴다.
3. **모델 지식 구분**: 검색으로 확인하지 못했지만 서술에 필요한 배경은 문장에 `(모델 지식)` 마커를 붙인다. 마커 없는 사실 주장은 sources로 뒷받침되어야 한다.
4. **세미나 맥락 필수**: `in_this_seminar`에 이 개념이 세미나 어디서 왜 등장했는지 슬라이드 태그 [D{n}-p{k}]와 함께 쓴다. 태그 규칙은 slide-interpretation 스킬과 동일.
5. 분량: 개념당 detail 4~10문장. 백과사전 전재가 아니라 학습자용 재서술.

## 출력 스키마
파일: `{폴더}/_workspace/08_concepts_{group:02d}.json`

```json
{
  "group": 1,
  "concepts": [{
    "id": "pruning",
    "name": "Pruning (프루닝)",
    "aliases": ["가지치기", "프루닝"],
    "category": "모델 압축",
    "easy_def": "한 줄 쉬운 정의",
    "detail": "원리·배경·역사·변형 기법을 4~10문장으로. 검색 확인 내용 기반, 미확인은 (모델 지식) 마커.",
    "in_this_seminar": "세미나 맥락 1~3문장. 슬라이드 태그 필수 [D1-p32].",
    "misconceptions": "흔한 오해/헷갈리는 점 (있을 때만)",
    "sources": [{"url": "검색으로 확인한 실제 URL", "title": "페이지 제목", "why": "무엇을 확인했는지"}],
    "related_concepts": ["quantization"],
    "web_verified": true
  }]
}
```

- `id`: 영문 소문자 kebab-case. 웹페이지 딥링크(`#concept-pruning`)와 해설 내 개념 링크의 키가 된다.
- `sources`: 항목당 1~4개. 논문 원문 > 공식 문서 > 신뢰할 수 있는 기술 블로그 순으로 우선한다.
- 저장 전 자기 검사: 슬라이드 태그 유효 범위, sources URL이 실제 검색/접속 결과인지, id 중복 여부.
